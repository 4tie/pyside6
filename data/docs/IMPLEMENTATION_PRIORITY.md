# IMPLEMENTATION_PRIORITY.md — What to Build First

## Build now
1. Environment and settings reliability
2. Download data flow
3. Backtest flow
4. Results UI quality
5. Logic-based analysis and suggestions
6. Strategy/config editing
7. Candidate version + diff + re-test flow
8. Compare / accept / rollback / history

## Build as placeholders only for now
1. Chat panel UI shell
2. AI provider settings shell
3. AI service interfaces
4. Disabled deep-analysis actions

## Build later
1. Provider integrations
2. Deep AI analysis
3. AI code editing
4. Natural-language app control

## Rule
If a change competes with core product work and AI integration work,
**core product wins by default**.
